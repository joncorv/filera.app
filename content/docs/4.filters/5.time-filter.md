# Time Filter

The **Time Filter** keeps or removes files based on a single date boundary — either everything before or everything after a chosen date.

## Controls

| Control | Description |
|---|---|
| **Date** | The reference date to compare against |
| **Filter Older Dates / Filter Newer Dates** | Toggle the direction of the filter |

## Modes

**Filter Older Dates:** Files whose modification date is older than (before) the chosen date are excluded. Only newer files remain active.

**Filter Newer Dates:** Files whose modification date is newer than (after) the chosen date are excluded. Only older files remain active.

## Behavior

- The comparison uses the file's last-modified timestamp
- Files modified exactly on the chosen date are treated as matching the boundary and are included

## Comparison with Time Period Filter

| Situation | Use |
|---|---|
| Files within a specific date range | [Time Period Filter](/docs/filters/time-period-filter) |
| Everything before a date | Time Filter (Filter Older Dates → keep newer, or Filter Newer Dates → keep older) |
| Everything after a date | Time Filter |

## Examples

Reference date: 2024-06-01

| Mode | Active files |
|---|---|
| Filter Older Dates | Files modified on or after 2024-06-01 |
| Filter Newer Dates | Files modified on or before 2024-06-01 |

## Tips

- Use this when you want a simple cutoff: "only rename files from this month" or "only rename files older than a year"
- For more precision, use [Time Period Filter](/docs/filters/time-period-filter) with both a start and end date
